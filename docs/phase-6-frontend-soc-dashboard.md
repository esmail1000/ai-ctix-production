# Phase 6 — Frontend SOC Dashboard

## Scope
Frontend-only SOC dashboard cleanup and wiring for real backend/API outputs from phases 1–5.

## Implemented changes

- Replaced the Threat Intel placeholder page with a report-scoped dashboard that reads real data through the secured threat-intel graph reader.
- Added report selection, CVE cards, NVD/CISA/MISP source status display, warnings/notes, and a negative CVE lookup link.
- Removed fixed trend percentages from the dashboard KPI and IOC panels.
- Replaced the synthetic attack graph drawing with a link to the real Neo4j graph page.
- Replaced dashboard attack-path construction with client-side loading from `/api/attack-paths/[reportId]`.
- Changed known-exploited CVE display to rely only on stored `knownExploited` or `cisaKev` flags.
- Made dashboard recommendation queue prefer backend recommendations/report remediation and display the source.
- Updated the recommendations page to display backend recommendation arrays and `recommendationSources` first, while clearly marking UI-derived fallback controls.
- Improved `/graph` metric calculation so graph stats can include impact/remediation nodes proven through returned attack paths.

## No-fake-data rules

- No fake attack path is rendered on the dashboard.
- No static graph nodes are drawn as if they came from Neo4j.
- No fixed trend percentages are displayed.
- No CVE is treated as known exploited unless stored `knownExploited` or `cisaKev` is true.
- No fake threat-intel counts, MISP status, CVSS, or KEV status are generated by the UI.
- UI-derived recommendation fallback is shown only when backend/report recommendations are unavailable and is labeled as fallback.

## Acceptance checks

1. `npm run build` passes.
2. `/threat-intel?reportId=<id>` shows CVE cards with NVD/CISA/MISP source statuses.
3. `/dashboard` shows no static trend percentages.
4. `/dashboard` attack path alerts load from `/api/attack-paths/[reportId]` or show an honest empty/error state.
5. `/dashboard` known-exploited CVEs are sourced from `knownExploited`/`cisaKev` only.
6. `/recommendations?reportId=<id>` shows backend recommendations and `recommendationSources` before fallback controls.
7. `/graph?reportId=<id>` continues to show real graph nodes/edges and attack-path evidence.
